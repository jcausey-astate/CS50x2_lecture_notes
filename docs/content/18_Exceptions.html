Exceptions

(When Everything Goes Wrong)


  • Used to communicate that something unexpected has occurred.
  • Allows the programmer to deal with unexpected situations in a controlled manner.
  • May be as simple or as complex as desired (depending on usage in the program)
  • In C++, any data value can be used as an exception!

Key Words


throw - Used to generate an exception and ‚Äúthrow‚Äù it up the call stack.

try ‚Äì Used to enclose a block of code in which an exception might occur (allowing the programmer to handle it if it does).

catch ‚Äì Used to detect and handle a specific type of exception, if it actually occurred during execution of a try block.

Exceptions are about Communication


When something goes very wrong in a way that you cannot fix in the local context, exceptions give you a way to communicate that problem to a higher level in the program structure.

Exceptions are:

  • out-of-band
    • They don‚Äôt add complexity to function or object interfaces.
      • They let the parameters and return types represent whatever they naturally should, without mixing in error signaling.

  • impossible to ignore
    • Someone using your library can‚Äôt just ignore them (and expect their program to keep running).
    • It forces us to think about what kinds of errors may happen.

Exceptions are a way to handle things that shouldn‚Äôt have happened when they inevitably do happen.

Flow of Control


When something unexpected happens, a function may choose to throw an exception.

  • At this point, execution leaves the function immediately!
  • Exception moved up the call stack until a try with a matching catch clause is found.
    • Execution resumes with the first line of the catch block (if found).
  • If not caught, the exception eventually makes it up to the OS, which will shut down the process.

Example


// function that *may* throw an exception
int totalDays(int days, int weeks){
    if((days < 0) || (days > 7)){
        // value being thrown is a C-string:
        throw "Invalid value for days.";
    }
    if(weeks < 0){
        throw "Invalid value for weeks.";
    }
    return 7 * weeks + days;
}

Example: Handling the Exception

int main(){
    int days, weeks;
    cout << "Enter days and weeks separated by space: \n";
    cin  >> days >> weeks;
    
    try {  
        int totDays = totalDays(days, weeks);
        cout << "Total days: " << totDays;
    }
    catch (const char* msg) {
        cout << "Error: " << msg;
    }
    return 0;
}

Should you rely on exception handling here?
    (NO, you should check the values first.)


But, exceptions provide an impossible-to-ignore failsafe in case the person writing main() wasn‚Äôt thinking about checking pre-conditions.

What Happens?

  • try block is entered. totalDays() function is called.
  • If 1st parameter is between 0 and 7, calculate the number of days and return as usual (no exception).
  • If 1st parameter is out of range, exception of type const char* is thrown: Execution leaves the function immediately.
  • try block execution is halted, execution jumps to first catch block, looking for a
    catch(const char*)
  • Since the type matches, execution resumes in the catch block.

Example: Exceptions Can‚Äôt be Ignored


int main(){
    int days, weeks;
    cout << "Enter days and weeks separated by space: \n";
    cin  >> days >> weeks;
    // No validity check at all!
    int totDays = totalDays(days, weeks);
    cout << "Total days: " << totDays;

    return 0;
}


This code does not check to see if the values entered are valid.

If they aren‚Äôt the exception will be thrown, but not caught, and the program will crash.


Exceptions cannot be (implicitly) ignored!

Example: Checking Pre-Conditions


int main(){
    int days, weeks;
    cout << "Enter days and weeks separated by space: \n";
    cin  >> days >> weeks;
    // check pre-conditions first:
    if( days >= 0 && days < 7 && weeks >= 0 ){
        int totDays = totalDays(days, weeks);
        cout << "Total days: " << totDays << '\n';
    }
    else{
        cout << "You entered invalid values.\n";
    }
    return 0;
}


This code checks the pre-conditions before calling totDays(), so the exception can‚Äôt be thrown.

This is probably the ‚Äúbest‚Äù (most well thought-out) implementation.

Notes

  • Many catch blocks may follow a single try (but there must be at least one).
  • You cannot catch without try, or try without catch.
  • catch blocks are scanned until one matches the type of the exception thrown.
  • catch blocks may or may not have formal parameter name (depending on whether the exception value itself will be needed inside the block).

Missed Exceptions

  • Failure to catch an exception just means it continues ‚Äúflying‚Äù up the call stack.
  • If it gets all the way out of main() without being caught, the program will be terminated by the OS.
  • It is possible to define a ‚Äúcatch-all‚Äù catch block (although it is often considered sloppy to do so):
    catch( ... )
    • If you define a ‚Äúcatch-all‚Äù, it should be the last catch in the chain.

Exception Classes/Objects

  • Any type may be thrown. Even custom objects.
  • We can define objects specifically for this purpose.
  • Exception classes are useful to put a meaningful name on the type (so that catch blocks are easy to understand).

Example

catch( NullPointerException )

is better than

catch( int )

Exception Classes

  • Don‚Äôt necessarily need members
    • Unless they do: if the added functionality is useful, add it!

Example

class NullPointerException{ };  // empty

Or:

class NullPointerException{
    public:
        string msg;            // with attribute
};

Standard Exceptions

The C++ Standard Library defines an exception base class in the <exception> header.

  • Useful (later) to make our own exceptions with nice common behaviors.
#include<exception>
using std::exception;

class NullPointerException : public exception{
    public:
        virtual const char* what()const {
            return ‚ÄúA null pointer was encountered.‚Äù;
        }
};

Documentation Guidelines


Exceptions require some new documentation‚Ä¶ Here‚Äôs an example:


/**
 * converts a numerical ranking in {1,2,3} to the corresponding 
 * medal color from {"gold","silver","bronze"}
 * 
 * @param  rank  integer numerical ranking; should be in {1,2,3}

Notice the @throw Doxygen tag ‚Äì it should be used once for each type of exception your function may throw.

Be sure to describe the exception type, as well as when/why it would be thrown.
