Abstract Data Types

When a built-in data type just isn‚Äôt enough‚Ä¶

  • Abstract Data Types are user-defined.
  • Can represent new ‚Äúthings‚Äù from the real world better than built-in types.
  • Essentially ‚Äúextend‚Äù the language.
  • Three mechanisms for creating ADTs in C++:
    • enum
    • struct
    • class

enum

Enumerated Type

  • Allows the programmer to specify mnemonic literals.
  • Low-level, not very flexible.
  • Example:
enum WorkDayAbr = {mon, tue, wed, thr, fri};

//... later in the program ...

WorkDayAbr today = mon;

enum (continued)

  • enum values are internally represented by integers.
  • enum will implicitly cast to int, but not vice-versa.
  • Limited to creation of custom literals (you cannot really create a complex data type using enum).
enum WorkDayAbr{ mon, tue, wed,
                 thr, fri };
-----------------------------------------
int rain[5]     {5, 15, 35, 95, 10};
string names[5] {"Monday", "Tuesday", "Wednesday", 
                 "Thursday","Friday"};
for(int i = mon; i <= fri; ++i){
    cout << "Rain chance for "
         << names[i] << ": " << rain[i]
         << endl;
}

Modern C++ prefers ‚Äúscoped enum‚Äù

enum class WorkDayAbr{ mon, tue, wed,   // struct or class
                       thr, fri };
-----------------------------------------
int rain[5]     {5, 15, 35, 95, 10};
string names[5] {"Monday", "Tuesday", "Wednesday", 
                 "Thursday","Friday"};
for (int i = static_cast<int>(WorkDayAbr::mon);
     i <= static_cast<int>(WorkDayAbr::fri); ++i) {
    cout << "Rain chance for " << names[i] << ": " << rain[i] << endl;
}
  • enum values do not implicitly cast to int - you have to static_cast.
  • You can define your own operators (like ++) to make it easier to work with these‚Ä¶
  • Core Guidelines: https://isocpp.github.io/CppCoreGuidelines/CppCoreGuidelines#S-enum

Structures

Structures are used to ‚Äúglue‚Äù several other types together to create a new type.

This is ideal when there is a lot of ‚Äúrelated‚Äù data that represents a single ‚Äúthing‚Äù in your code.

  • Structures specify attributes, and potentially also behaviors.
  • The keyword struct is used to define a structure:
struct Circle {
   int x;
   int y;
   double radius;
};  // Notice the semicolon!

Structures (continued)

  • The variables declared in a structure definition are called attributes (or sometimes ‚Äúmembers‚Äù).
  • You can create a variable (or instance) of the new type:
Circle unitCircle;
  • Then you can access its attributes using ‚Äúdot notation‚Äù:
unitCircle.x      = 0;
unitCircle.y      = 0;
unitCircle.radius = 1.0;

Structures (continued)

  • The member access operator (AKA the ‚Äúdot operator‚Äù):     .

  • Initialize with C++11‚Äôs uniform intializer syntax:

Circle anotherCircle{2, 3, 3.8};
  • Notice that the order of items in the list matters!
  • You can find out the size of a structure using sizeof():
cout << sizeof(anotherCircle);  // using an instance
// ... or ...
cout << sizeof(Circle);         // using the type

Legacy C++ code may use the following syntax to initialize a structure:

Circle yetAnotherCircle = {3, 2, 2.4}; // old style, deprecated

Structures (continued)

Structures create a new type!

Do not expect built-in functions or operators to work with the new type by default:

Circle myCircle{1, 2, 3.1};
cout << myCircle << endl;   // Will NOT work!


You can define your own functions:

void printCircle(Circle c){
    cout << "("      << c.x
         << ","      << c.y
         << ") r = " << c.radius;
}

Structures and Functions

The previous function passed a structure to the printCircle() function by value.

  • Is this a good idea?
  • Consider passing by (const) reference instead:
void printCircle(const Circle& c){
    cout << "("      << c.x
         << ","      << c.y
         << ") r = " << c.radius;
}

And if you need the parameter to be changed by the function, use non-const reference:

void readCircle(Circle& c){
    cout << "Enter x and y: ";
    cin  >> c.x >> c.y;
    cout << "Enter radius:  ";
    cin  >> c.radius;
}

Parallel Arrays VS Structures

  • Parallel arrays store related data by common indexing.
  • Structures store related data in a single variable‚Ä¶
    • An array of structures can be used in place of parallel arrays.
  • When should you choose structures?
    • How closely are the data related?
    • Do the items really ‚Äúbelong‚Äù to the same thing?
      • If so, structures would make sense
  • Passing a single array VS passing several arrays
    • Ease of use is always a factor!

What About Operators?

A look ahead

You can also overload operators (and other functions) so that they will work with your new type:

std::ostream& operator<< (std::ostream& outstr,
                          const Circle& c) {
    outstr << "("      << c.x
           << ","      << c.y
           << ") r = " << c.radius;
    return outstr;
}

Assignment

What happens when you assign one structure to another directly?

  • The assignment operator ( = ) makes a bit-wise copy of the entire structure‚Äôs contents.
    • ‚ÄúPlain old data‚Äù is copied exactly correctly.
    • Arrays are encapsulated in the structure; they get copied as well!
      • Contrast with how arrays are usually treated‚Ä¶
  • Pointers can cause trouble!
    • The copy applies to the pointer only, not the item being pointed to.
  • Resources can cause trouble!
    • Never make a copy of something like an open stream.

Initial Values

C++11 added the ability to include initialization in a structure definition:

struct Person {
    int    age   = 0;    // initialize to 0
    string name;
};


Here, inline member initialization is used to provide initial values for age.


Person p3{20, "Cassandra"}; // only if -std=c++14 or later

C++ version warning:
If you specify initializers, you will not be able to use uniform initializer syntax with your new type in C++11, but in C++14 and afterward, it is OK.

(optional) Initial Values (Uniform Syntax)

You can also set initial values using uniform initializer syntax:

struct Person {
    int    age{0};         // initialize to 0
    string name;
};


Here, uniform initializer syntax is used to provide initial values for all members of both structures.


Person p3{20, "Cassandra"}; // only if -std=c++14 or later
